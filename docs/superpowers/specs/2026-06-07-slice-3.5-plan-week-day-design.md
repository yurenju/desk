# Slice 3.5 — Plan 週導覽 + Day 欄可規劃（設計）

> 對應 [ROADMAP.md](../../../ROADMAP.md) 的「Slice 3.5 — Plan 週導覽 + Day 欄可規劃」。本文把該段落落實為可實作的設計。

## 目標

把 Plan 模式從「選取日寫死今天、Day 欄唯讀」變成**三欄（Month / Week / Day）繞單一「焦點日」耦合、且 Day 欄完全可互動**：

1. **可選哪一天 promote**：焦點日進 URL，可書籤；Week 欄日格可點換焦點日、`‹ ›` 換週。
2. **Plan 裡能排該天的每日三件事**：Day 欄解除唯讀，設 `daily_priority`、勾完成、編輯、刪除、新增——與 Focus 的 Day 欄同等能力。

**Plan 與 Focus 的本質區分**（沿用 ROADMAP，兩 tab 非冗餘）：Focus 把大尺度資訊**降權**（執行畫面，某日 hero）、Plan **升權**（規劃畫面，月/週/日並列）；同樣的互動能力、不同的視覺權重與時機。**Focus 現有版型不動。**

## 範圍邊界

**做**：route `/plan/$month` → `/plan/$date`、Week 欄導覽、Day 欄互動解耦、store action 吃焦點日、Month 欄 `‹ ›` 改跳月、Today→Focus 標籤改名（route 不動）。

**不做 / 延後**（沿用 ROADMAP）：

- 真實拖曳、跳回今天鈕、過場動畫、鍵盤導覽 → Slice 7。
- 三欄在 Plan 的視覺權重微調、promote 後是否自動跳焦點日 → 做完實際用一輪再列。
- 跨月遠期日的特別處理 → 不做（遠期留月層，該週到了再排日）。

## 設計決策

### 決策 1：焦點日怎麼餵進 store actions（採「動作加顯式 `date` 參數」）

現況問題：store 的 `addTodayTask`、`setDailyPriority` 在 wrapper 層硬塞 `store.today`，導致 Plan 的 Day 欄無法對非今天的焦點日操作。

底層 pure helper **本來就吃日期參數**——`addTodayTask(prev, title, today, id, now)`、`setDailyPriority(prev, id, n, today)`，只是 call site 把 `today` 寫死。

採用方案：**把 store wrapper action 也改成吃 `date` 參數**，由呼叫端（Focus 傳其檢視日、Plan 傳焦點日）傳入。

- ✅ 與現有 pure-function 風格一致、好測、無隱藏全域狀態。
- 否決「store 存 `focusDate` 全域狀態」：Focus 與 Plan 共用同一 store，切 tab 易 stale。
- 否決「Plan 暫時覆寫 `store.today`」：污染「今天」的真實語意。

**附帶修復一個 latent bug**：Focus 用 `/today/$date` 檢視非今天的某日時，現況 `setDailyPriority`/`addTodayTask` 仍 scope 到真實今天（騰位算錯天、新增排錯日）。thread 檢視日後一併修正。

### 決策 2：promote 不需改 action

Slice 3 的 promote 已經吃 `selectedDate`（`useMonthRow(id, { month, selectedDate })`，label 已是 `→ 排到 N 日`）。它只排到今天純粹因 `plan.tsx` 把 `selectedDate` 寫死。**route 修好，promote 自動跟焦點日**，不需動 promote 邏輯。

### 決策 3：`DayColumn` 互動性與 variant 解耦

現況 `const interactive = variant === "today-hero"` 把互動性綁死版型。改為獨立 `interactive` prop：Plan 傳 `variant="plan-narrow" interactive`，維持窄版視覺但開放互動。

### 決策 4：三欄繞單一焦點日（為何不做限本週小版本）

沿用 ROADMAP：曾考慮「選取日」與「月份」各自獨立、限本週可選，但最終是「月/週/日繞單一焦點日耦合」，獨立版會變技術債。週為主完整模型只比「讓日格可點」多一個「Week `‹ ›` 換週」，故直接做完整版。

## 架構與資料流

```
URL /plan/$date  (焦點日，single source of truth)
        │
        ▼
   PlanView({ date })
        │  month = currentMonthISO(date)
        ▼
   PlanLayout({ allTasks, selectedDate: date, month })
        ├── MonthColumn   ‹ › → addMonths(date, ∓1)  ；promote → selectedDate
        ├── WeekColumn    日格 → 該日 ；‹ › → addDays(date, ∓7)
        └── DayColumn(variant="plan-narrow", interactive)
                └── Top3Card / TaskRow / AddTaskInput  ← 全部 thread selectedDate
                        └── useTaskRow(id, date) → setDailyPriority(id, n, date)
                        └── addTodayTask(title, date)
```

三欄的月份與週都由焦點日推導，URL 是唯一真實來源；任何導覽動作都是「換一個 `/plan/$date`」。

## 元件層改動

### Route

| 檔案 | 改動 |
|---|---|
| `src/routes/plan.$month.tsx` → `plan.$date.tsx` | param `$month` → `$date`；`beforeLoad` 用 `isValidDateParam`（照抄 `today.$date.tsx`）；非法 redirect `/plan` |
| `src/routes/plan.index.tsx` | 預設 `PlanView date={todayISO()}`（取代 `currentMonthISO()`） |
| `src/routes/plan.tsx` `PlanView` | 簽名 `{ date }`；`month` 由 `currentMonthISO(new Date(date + "T00:00:00"))` 推導；傳 `selectedDate={date}`（不再寫死 `today`） |
| `src/routeTree.gen.ts` | 重新產生 |

### `src/lib/date.ts` 新增

```ts
// Shift an ISO date (YYYY-MM-DD) by n days.
export function addDays(date: string, n: number): string

// Shift an ISO date by n months, clamping the day to the target month's last day
// (e.g. addMonths("2026-01-31", 1) === "2026-02-28").
export function addMonths(date: string, n: number): string
```

### Store（`src/store/tasks.ts` + 介面）

```ts
// before → after
addTodayTask: (title: string) => Promise<void>
  → addTodayTask: (title: string, date: string) => Promise<void>

setDailyPriority: (id: string, n: Priority | null) => Promise<void>
  → setDailyPriority: (id: string, n: Priority | null, date: string) => Promise<void>
```

- wrapper 內把 `today` / `get().today` 換成傳入 `date`，傳給既有 pure helper（`taskOps.ts` 的 pure 簽名不動）。
- promote 相關不動。

### 互動 wiring

| 檔案 | 改動 |
|---|---|
| `src/features/day/DayColumn.tsx` | 加 `interactive?: boolean` prop，刪 `const interactive = variant === "today-hero"`；`interactive ??= variant === "today-hero"` 作為預設以維持 Focus 行為；把 `selectedDate` thread 給 `Top3Card`/`TaskRow`/`AddTaskInput` |
| `src/features/day/useTaskRow.ts` | `useTaskRow(id)` → `useTaskRow(id, date)`；`setPriority` 改呼叫 `setDailyPriority(id, n, date)` |
| `src/features/day/TaskRow.tsx` / `Top3Card.tsx` | 吃並下傳 `date`（焦點日）給 `useTaskRow` |
| `src/features/day/AddTaskInput.tsx` | 加 `date` prop，呼叫 `addTodayTask(value, date)`；placeholder 改中性文案 |
| `src/features/plan-view/PlanLayout.tsx` | `DayColumn` 改傳 `variant="plan-narrow" interactive` |
| `src/features/plan-view/TodayLayout.tsx` | 把檢視日 thread 給 `DayColumn`（修 latent bug；Focus 仍 `variant="today-hero"`） |

### Week 欄（`src/features/week/WeekColumn.tsx`）

- 日格 `<li>` 包成 `<Link to="/plan/$date" params={{ date }}>` → 點擊換焦點日；選中日維持現有 `styles.selected`。
- header 加 `‹ ›` 兩個 `<Link to="/plan/$date" params={{ date: addDays(selectedDate, ∓7) }}>`。

### Month 欄（`src/features/month/MonthColumn.tsx`）

- 兩個 `‹ ›` 從 `to="/plan/$month" params={{ month: prevMonth/nextMonth }}` 改為 `to="/plan/$date" params={{ date: addMonths(selectedDate, ∓1) }}`。

### Focus 改名（只動標籤／文案，route 維持 `/today`）

- TopNav「Today」分頁標籤 → 「Focus / 專注」。
- `AddTaskInput` placeholder「加一件今天的事」→ 中性如「加一件這天的事」。
- `DayColumn` header eyebrow 的「今天」字樣：依 `selectedDate` 是否 == 真實今天決定顯示，非今天時不顯示「今天」。

## 錯誤處理

- route param 非法（非 `YYYY-MM-DD`）→ `beforeLoad` redirect `/plan`（沿用 Focus 既有模式）。
- mutation 失敗（新增/設優先權）→ 沿用現有 patch queue + 樂觀更新 + 失敗 toast/reload，焦點日不改變既有行為。
- 焦點日落在當月以外不特別處理：Month 欄顯示焦點日所屬月、Week 欄顯示焦點日那一週，三者一致即可。

## 測試策略

### 自動化（vitest / Testing Library）

- `addDays` / `addMonths`：純函式邊界（跨月、跨年、月底 clamp）。
- `setDailyPriority(id, n, date)`：對**非今天**的焦點日騰位，scope 只動該日的 task。
- `addTodayTask(title, date)`：排到指定日（`scheduled_dates = [date]`）。
- `DayColumn` `interactive` 與 `variant` 解耦：`variant="plan-narrow" interactive` 仍渲染 ring / checkbox / AddTaskInput；`today-hero` 預設行為不變。
- route：`plan.$date` 的 `beforeLoad` 非法 param redirect。
- 既有 `taskOps.test.ts` / `tasks.test.ts` / `TaskRow.test.tsx` / `useTaskRow.test.ts` 因簽名改動需同步更新（補 `date` 引數）。

### e2e（`npm run test:e2e`，Playwright）

> CLAUDE.md 規定：改 Today/Plan 互動、登入流程、使用者操作流程後，除 vitest 也要跑 e2e。本片改 Plan 互動，必跑並同步更新 `e2e/*.spec.ts`。

- Plan 點 Week 日格 → URL 變 `/plan/<該日>`、Day 欄與 Month 欄隨之更新。
- Plan `‹ ›` 換週（焦點日 ±7）、Month `‹ ›` 換月（焦點日 ±1 月）。
- Plan Day 欄設每日三件事（ring）+ 勾完成 + 新增一件 → 落到焦點日。
- Month 欄 promote → 排到焦點日（非寫死今天）。

### 手動測試（preview + AI agent）

> CLAUDE.md 規定：視覺/互動打磨需 AI agent 透過 `preview_start` 實際操作驗收。先啟動 preview 探測登入狀態——已登入就直接開始驗，未登入且驗收需真實資料時才請使用者協助 device flow（WSPC 端 Approve）。

- 三欄繞單一焦點日的整體手感：點日格 / 換週 / 換月後三欄是否一致。
- 窄版 Plan Day 欄互動目標 ≥44px（touch）、ring / checkbox / 新增輸入可用。
- 對照下列「驗收標準」逐項手動驗收。

## 驗收標準

1. 進 `/plan` 預設焦點日為今天；`/plan/2026-06-10` 直接開在該日，可書籤。
2. 非法日期 `/plan/foo` redirect 回 `/plan`。
3. Week 欄日格可點，點下後 URL、Month 欄、Day 欄三者同步切到該焦點日。
4. Week 欄 `‹ ›` 把焦點日 ±7 天（落在同星期幾），三欄同步。
5. Month 欄 `‹ ›` 把焦點日 ±1 月（日期 clamp），三欄同步。
6. Plan 的 Day 欄可設 `daily_priority`（三件事 ring）、勾完成、編輯、刪除、新增；新增與騰位都作用在**焦點日**而非今天。
7. Month 欄 promote 排到**焦點日**（非寫死今天）。
8. Focus（`/today`）行為完全不變；其 Day 欄仍 hero 版型、互動如舊。
9. Focus 用 `/today/<非今天>` 檢視某日時，設優先權/新增正確作用在該日（latent bug 修復）。
10. TopNav 分頁標籤改為「Focus / 專注」；AddTaskInput 文案中性化；route 仍為 `/today`。
11. 全部自動化測試 + `npm run build` 型別 + e2e 通過。
